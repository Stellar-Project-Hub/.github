# Contributing to Stellar Project Hub

First off, thank you for being here! We are excited to build the future of the Stellar ecosystem with you through the **Drips Wave** program. To ensure a smooth experience for everyone, please follow these guidelines.

## 🌊 The Drips Wave Workflow

To participate, follow these steps:

### 1. Find and Claim an Issue
- Browse the **Issues** tab in any of our repositories via the [drips app](https://www.drips.network/app)..
- Open issues are eligible for points.
- **To Claim:** Comment on the issue. Once a maintainer assigns it to you, you are clear to start. Do not start work until you are assigned.

### 2. Development Environment
- **Fork** the repository to your own GitHub account.
- **Clone** your fork locally.
- Create a new branch for your task: `git checkout -b feature/issue-number-short-description`.

### 3. Standards & Quality
- **Code Style:** Follow the existing patterns in the codebase.
- **Commits:** Use descriptive commit messages (e.g., `feat: add soroban auth logic for vault`).
- **Documentation:** If you add a feature, update the relevant `README.md` or inline docs.

### 4. Submitting Your Work
- Push your changes to your fork.
- Open a **Pull Request (PR)** against our `main` branch.
- Reference the issue number in your PR description (e.g., `Closes #12`).
- Ensure all CI/CD checks (if any) are passing.

### 5. Review & Approval
- A maintainer will review your PR within 24–48 hours.
- Address any requested changes promptly.
- Once merged, your contribution is complete, and points will be awarded via the Drips platform.

## 🚫 What Not To Do
- Do not submit PRs for issues that have not been assigned to you.
- Do not group multiple unrelated fixes into a single PR.
- Avoid "low-effort" PRs (e.g., just fixing a single typo unless it's a documentation issue).

## 💬 Need Help?
If you have questions about a specific task, please comment directly on the **GitHub Issue** or start a thread in the **Discussions** tab.

**Happy Coding!**
